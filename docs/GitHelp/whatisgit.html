How to use Git

What is Git?

Git is a version control system.

What is a version control system?

A version control system is a system that records changes to a file or set of files.

This allows you to roll back to specific versions of those files later, not just the most recent versions.

How to set up Git?

Windows

Firstly, I don't personally recommend you use Windows for programming. If you do use it though, this is how to set up Git.

  1. Download Git for Windows
  2. Install the file.
  3. Search for Git in your Programs. You will see both Git GUI and Git Bash. We will focus on using Git Bash, since the GUI is pretty straightforward.
  4. You are done installing Git on Windows!

Linux

  1. Open Terminal (Keyboard shortcut is Ctrl+Alt+T on Ubuntu)
  2. Type the following code: (and enter your password when prompted)
    sudo apt-get update
    sudo apt-get install git
  3. Done!

Mac OS X

  1. Download Git for Mac
  2. Install the file.
  3. Finished

There are a number of important commands for Git. You enter them by typing them into the bash console, or the terminal. This tutorial will walk you through cloning a repository, adding a new file, committing the new file, then pushing your changes to the repository.
This is the repository we will be using.

Cloning a repository: git clone

git clone is the first thing you do when joining a new repository for this tutorial.

  1. In your terminal, make a directory where you want to put your project.
    mkdir theprogrammingclub
  2. Navigate to that directory.
    cd theprogrammingclub
  3. Now, click on this link to go to the site on github.com, you may need to sign in or create an account if you haven't.
  4. On the right side of the screen there is a box that says HTTPS clone URL, as seen below.
  5. Copy the link to your clipboard.
  6. In your terminal screen, type git clone
  7. Now right click the terminal screen, and press paste.
  8. Press enter.
  9. If all went according to plan, (you may need to put in your Github info) you should now have the repository in your 'theprogrammingclub' folder!

Adding a new file: git add

After having made a new file, we will then want to add that file to our git index. It is very straightforward to do so.

  1. Make a text file, call it yourname.txt. In my case, it would be joshbaker.txt
  2. Open it and write "yourname is doing the git tutorial!"
  3. Save the file to theprogrammingclub directory, in the repository you just cloned, then close the file
  4. In your terminal, navigate to the repository.
    cd
    cd theprogrammingclub
    cd SampleRepository
  5. We want to add the file to our git index. To do so, type:
    git add yourname.txt
  6. We have now added the file, and should commit it to the local repository.

Committing a new file: git commit

After having made a new file, we will then want to commit that file to our local repository.

  1. First we want to make sure there have been no changes to the repository since we last cloned. To do so, type:
    git pull
  2. Now, type
    git commit
  3. Type the message you want to be with your commit. This will be information about what you have done.
  4. Press enter
  5. Your commit should have succeeded!
  6. (if you don't want to go through the effort of adding then committing, simply typing git commit -a should add then commit all files)

Pushing a new file: git push

The final step to adding a file to a repository. This brings all of our changes from the local repository to the remote repository

  1. While still in the same directory, now type:
    git push
  2. Your code should now be pushed! Go to github.com to see if your code is there.
